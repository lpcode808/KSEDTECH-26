How this app works

A non-technical walkthrough explaining the architecture, code choices, and offline mechanics of the Kamehameha Schools Ed Tech Conference schedule guide.

Project CONNECT26 App
Target Audience Educators & Students
Stack HTML5, Vanilla CSS, JS
Offline-Ready Yes (PWA Service Worker)

01 What is this app?

This is a progressive web app (PWA) built to serve as a high-speed, offline-capable companion for attendees of the 2026 Kamehameha Schools Education Technology Conference (CONNECT26) at the Neal S. Blaisdell Center.

Instead of packing a heavy, general-purpose app framework that forces you to download megabytes of code, this is a custom-coded single-file client interface. It loads instantly, filters through all 55+ sessions in real-time as you type, and lets you take private notes on your own device.

A "Vanilla" Philosophy: In the web development world, "vanilla" means plain, pure code written without frameworks like React, Angular, or Vue. We chose vanilla because it eliminates all build-step overhead, makes the codebase easy for anyone to read or modify, and performs flawlessly on older mobile devices.

02 The big picture: No accounts, no database, 100% private

Almost every modern app forces you to create an account, log in, and send your activity back to a corporate cloud database. This app does the opposite. There is no server, no database, and no login required.

The schedule data and speaker details are baked directly inside the application files. When you type in a personal note or click the star next to a speaker, that data is saved exclusively inside your browser's private sandbox. Your notes and favorite speakers are never transmitted across the network, making this app 100% private, instant, and entirely Yours.

03 The files: intentionally tiny

The entire application consists of just three small, essential files sitting in a folder:

  1. index.html — The Skeleton and the Data: Contains the structural layout, all 55+ sessions in an embedded data list, the CSS styles for the beautiful theme, and the script logic that powers the tabs and interactive features.
  2. sw.js — The Service Worker: A small helper script that acts as an offline proxy, telling the device to save the app files so it runs when cell reception is dead.
  3. favicon.svg — The Logo: A tiny, scalable graphic that appears in your browser tab or on your device's home screen.

04 How offline mode works (The Service Worker)

The Neal S. Blaisdell Center is notorious for dropping cell coverage and leaving Wi-Fi networks congested during peak conference times. To ensure attendees can always access their schedule and notes, the app uses a browser capability called a Service Worker.

A Service Worker is like a clever background butler. The first time you open the page while online, the Service Worker intercepts the request, grabs the app files, and tucks them away in a secure drawer on your device called the Cache Storage.

The next time you pull up the app—even if you're in a cellular dead zone—the background butler steps in before the browser can say "no connection" and immediately serves the cached files. The app loads instantly, offline, and ready to go.

// Inside sw.js — caching the app so it works inside the venue
const CACHE_NAME = 'connect26-guide-v2';
const ASSETS = [
  './',
  './index.html',
  './favicon.svg'
];

05 How your notes and stars are saved (localStorage)

If there's no cloud database, how does the app remember which speakers you starred or what notes you typed when you close your browser tab or reboot your phone?

It uses localStorage—a built-in key-value vault provided by your web browser. Think of it like a private notebook that the browser keeps for this specific website. When you star a speaker or edit a note, the JavaScript code writes a tiny snippet of structured text to this vault:

// How your browser holds your starred speakers
localStorage.setItem('connect26_stars', JSON.stringify(['speaker-12', 'speaker-34']));

Because this vault resides directly on your physical hardware, reading and writing to it takes less than a millisecond. To prevent accidental data loss when switching browsers or clearing your history, the app includes a prominent Export button that lets you download your personal notes as a standard plain-text Markdown file.

06 Dynamic Search: Fast and local

When you type a query in the search bar, traditional sites send that search word back to a server, wait for a database query, and refresh the page. That loop takes several seconds and breaks if you go offline.

In our app, search is fully local. A lightweight search routine is triggered on every keypress. It parses the entire 55-session catalog instantly using simple JavaScript matching:

// Instant search logic inside index.html
const matchesSearch = 
  s.title.toLowerCase().includes(query) || 
  s.speaker.toLowerCase().includes(query) || 
  s.description.toLowerCase().includes(query);

This local matching evaluates all sessions in less than 5 milliseconds, hiding or revealing cards instantly as you type.

07 Sharing with QR codes (Purely Client-Side)

The "Share this app" button at the bottom pops up a QR code modal. Many sites generate QR codes by hitting a web API (like Google Charts) or loading a heavy external library.

To keep the application light and fully independent, we embedded a simple pure client-side QR generator directly inside the script. It mathematically calculates the pixel grid of the QR code using a tiny, self-contained SVG generation function, meaning you can open the share modal and let a colleague scan your screen even when both of you are completely disconnected from the internet.

08 Glossary

  • Client-Side — Code that runs directly on your device (computer or phone) rather than on a remote server.
  • Static Site — A site composed of static files (HTML, CSS, JS) that are sent to your browser exactly as they are stored, without dynamic processing on the hosting server.
  • Vanilla JavaScript — Pure JavaScript code written without relying on external libraries or frameworks.
  • PWA (Progressive Web App) — A website that behaves like a native mobile application, utilizing modern browser APIs to support home-screen installation and offline capabilities.
